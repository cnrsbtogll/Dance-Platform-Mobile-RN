Introduction

Dancer Community is a comprehensive mobile application that connects dance students with professional instructors. Built with React Native, TypeScript, and Expo, it provides a seamless experience for both students and instructors.

Project Overview

The application supports multiple dance styles including:

  • Salsa
  • Bachata
  • Kizomba
  • Tango
  • Modern Dance

Technology Stack

Framework: React Native 0.74.5
Platform: Expo SDK 51
Language: TypeScript 5.3.3
State Management: Zustand 5.0.8
Navigation: React Navigation 7.x
i18n: i18next & react-i18next

Getting Started

Prerequisites

Before you begin, ensure you have the following installed:

  • Node.js v18.0.0 or higher
  • npm v9.0.0 or higher
  • Expo CLI (install globally: npm install -g expo-cli)
  • For iOS: Xcode 14.0+ (Mac only)
  • For Android: Android Studio with Android SDK

Installation

  1. Navigate to the project directory
  2. Install dependencies: 
    npm install
  3. For iOS (Mac only), install CocoaPods: 
    cd ios
pod install
cd ..

Running the Project

Development Mode

npm start

iOS Simulator

npm run ios

Android Emulator

npm run android

Web Browser

npm start --web

Branding Your App

Before diving into development, you should customize your app's branding elements. This includes the app name, logo, splash screen, and other visual assets.

App Name

To change your app name, you need to update the configuration in app.config.js:

  1. Open app.config.js in the root directory
  2. Locate the brands object
  3. Update the name property in the codecanyon brand object: 
    const brands = {
  codecanyon: {
    name: 'Your App Name', // ✅ Change this - Update your app name here
    slug: 'your-app-slug',
    // ...
  }
};

Important: Update the name property inside the codecanyon brand object.

Note: The app name will also appear in the About screen. You can customize it further in the translation files (src/locales/tr.json and src/locales/en.json) under the about.appNameCodecanyon key.

App Icon

To replace the app icon:

  1. Prepare your icon files:
    • iOS: 1024x1024px PNG (no transparency)
    • Android: 1024x1024px PNG (no transparency)
    • Adaptive Icon (Android): 1024x1024px PNG with safe zone (foreground should be centered in 768x768px area)
  2. Replace the icon files in the assets/ folder:
    • assets/icon.png - Main app icon
    • assets/adaptive-icon.png - Android adaptive icon
  3. Update app.config.js to reference your icon files (if using different filenames): 
    const brands = {
  codecanyon: {
    icon: './assets/icon.png', // Your icon file
    adaptiveIcon: './assets/adaptive-icon.png', // Your adaptive icon
    // ...
  }
};

Best Practices:

  • Use a square icon with rounded corners (the system will apply the appropriate mask)
  • Keep important content within the center 80% of the icon
  • Use high-resolution images (1024x1024px minimum)
  • Avoid text in icons (it may be too small to read)

Splash Screen

The splash screen is the first screen users see when launching your app. To customize it:

  1. Create your splash screen image:
    • Recommended size: 1242x2436px (iPhone X and newer) or 2048x2732px (iPad)
    • Format: PNG with transparency support
    • Content: Your logo or app branding centered on the screen
  2. Save your splash screen to the assets/ folder:
    • assets/splash.png - Your splash screen
  3. Update app.config.js: 
    const brands = {
  codecanyon: {
    splash: './assets/splash.png', // Your splash screen
    // ...
  }
};

// In the expo config:
splash: {
  image: currentBrand.splash,
  resizeMode: 'contain',
  backgroundColor: '#ffffff', // Change this to match your design
}
  4. Customize the background color in app.config.js to match your splash screen design

Note: The splash screen also appears in the About screen. Make sure your splash image is appropriate for both contexts.

Favicon (Web)

For web builds, you'll need a favicon:

  1. Create a favicon file:
    • Size: 32x32px or 48x48px
    • Format: PNG or ICO
  2. Save it to assets/favicon.png
  3. The configuration in app.config.js will automatically use it: 
    web: {
  favicon: currentBrand.favicon,
}

About Screen Logo

The About screen displays your splash screen image as the app logo. To customize it:

  1. Update src/screens/shared/AboutScreen.tsx: 
    // The About screen uses the splash image
<Image
  source={require('../../../assets/splash.png')}
  style={styles.logoImage}
  resizeMode="contain"
/>
  2. If you want to use a different image for the About screen, replace the require() path with your custom logo file

Lesson Images

To customize lesson images displayed throughout the app:

  1. Add your lesson images to the assets/lessons/ folder
  2. Update src/screens/instructor/CreateLessonScreen.tsx and src/screens/instructor/EditLessonScreen.tsx: 
    // In CreateLessonScreen.tsx and EditLessonScreen.tsx
const LESSON_IMAGES = [
  require('../../../assets/lessons/salsa-1.jpg'),
  require('../../../assets/lessons/salsa-2.jpg'),
  require('../../../assets/lessons/bachata-1.jpg'),
  // Add your custom lesson images here
];
  3. Update the imageUrl values in src/data/lessons.json to match your image filenames

Recommended Image Specifications:

  • Aspect Ratio: 16:9 (recommended for lesson cards)
  • Minimum Resolution: 800x450px
  • Format: JPG or PNG
  • File Size: Keep under 500KB for optimal performance

Bundle Identifier and Package Name

To change your app's bundle identifier (iOS) and package name (Android):

  1. Open app.config.js
  2. Update the bundleIdentifier (iOS) and package (Android) for your brand: 
    const brands = {
  codecanyon: {
    bundleIdentifier: 'com.yourcompany.yourapp', // iOS
    package: 'com.yourcompany.yourapp', // Android
    // ...
  }
};
  3. Important: Once you publish to app stores, you cannot change these identifiers. Choose them carefully!

Naming Convention:

  • Use reverse domain notation: com.yourcompany.appname
  • Use lowercase letters and numbers only
  • No spaces or special characters
  • Must be unique (check availability before publishing)

Translation Files (App Name in UI)

To update the app name displayed in the UI (About screen, etc.):

  1. Open src/locales/tr.json and src/locales/en.json
  2. Update the about.appNameCodecanyon key: 
    {
  "about": {
    "appNameCodecanyon": "Your App Name",
    // ...
  }
}
  3. Update about.copyrightCodecanyon to include your app name: 
    {
  "about": {
    "copyrightCodecanyon": "© 2024 Your App Name. All rights reserved."
  }
}

Architecture

Project Structure

src/
├── components/      # Reusable React components
├── screens/         # Screen components
├── navigation/      # Navigation configuration
├── data/           # Mock data JSON files
├── services/       # Service layer
├── store/          # Zustand state management
├── types/          # TypeScript type definitions
└── utils/          # Utility functions

State Management

The application uses Zustand for state management. Key stores include:

  • useAuthStore: User authentication and session management
  • useLessonStore: Lesson data and operations
  • useBookingStore: Booking management
  • useNotificationStore: Notification handling
  • useProfileStore: Profile editing state
  • useThemeStore: Theme and language preferences

Navigation Structure

The app uses a nested navigation structure:

  • RootNavigator: Top-level navigator switching between Student and Instructor modes
  • StudentNavigator: Stack navigator for student screens with bottom tabs
  • InstructorNavigator: Stack navigator for instructor screens with bottom tabs

Data Flow

  1. User interactions trigger actions in components
  2. Actions update Zustand stores
  3. Stores notify subscribed components
  4. Components re-render with new data
  5. Mock data service provides data layer abstraction

Features Documentation

Student Module

Home Screen

The student home screen displays:

  • Personalized greeting with user's name
  • Category filter chips (All, Salsa, Bachata, Tango, Kizomba, Modern)
  • Filter button for advanced filtering (price, rating, duration)
  • Lesson cards with image, title, instructor, price, rating, and duration
  • Notification badge in header

Lesson Discovery

Students can:

  • Browse lessons by category
  • Filter by price range, minimum rating, and maximum duration
  • View lesson details including description, schedule, and instructor info
  • See reviews and ratings from other students

Booking Flow

  1. Select a lesson from the home screen
  2. View lesson details (LessonDetailScreen)
  3. Click "Derse Kaydol" (Register for Lesson)
  4. If not logged in, redirect to LoginScreen
  5. After login, navigate to PaymentScreen
  6. Complete payment (currently mock)
  7. Booking is created and added to "My Lessons"

My Lessons

Students can view:

  • Upcoming lessons (active tab)
  • Past lessons (past tab)
  • Lesson details, instructor contact, and cancellation options

Instructor Module

Dashboard

The instructor dashboard displays:

  • Statistics: Active lessons, total students, average rating, total earnings
  • Upcoming lessons with date, time, and student count
  • Quick actions: Create lesson, view all lessons
  • Earnings overview with "View Details" button

Instructors can create lessons with:

  • Lesson image selection (from predefined set)
  • Basic information: title, description, category, price
  • Duration selection (30, 60, 90, 120 minutes)
  • Date and time scheduling
  • Currency selection (USD, EUR, TRY)

Lesson Management

Instructors can:

  • View all lessons (active and past)
  • Edit existing lessons
  • Delete lessons (planned)
  • View student count and average rating per lesson

Earnings

Instructors can:

  • View total earnings on dashboard
  • Access detailed earnings screen with graph
  • See earnings breakdown by date
  • Filter earnings by time period

Both students and instructors can:

Notifications

Users receive notifications for:

Profile Management

Customization Guide

Theme Customization

Edit src/utils/theme.ts to customize:

  • Color schemes (student/instructor, light/dark)
  • Spacing values
  • Typography settings
  • Border radius values

Example: Changing Primary Color

export const colors = {
  student: {
    primary: '#005f73', // Change this value
    // ...
  }
};

Adding New Languages

  1. Create a new JSON file in src/locales/ (e.g., de.json)
  2. Copy the structure from en.json
  3. Translate all values
  4. Update src/utils/i18n.ts: 
    import de from '../locales/de.json';

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: en },
    tr: { translation: tr },
    de: { translation: de }, // Add new language
  },
  // ...
});

Modifying Mock Data

To modify mock data:

  1. Open the relevant JSON file in src/data/
  2. Edit the data structure
  3. Save the file
  4. The app will automatically reload with new data

Adding New Screens

  1. Create the screen component in the appropriate folder:
    • src/screens/student/ for student screens
    • src/screens/instructor/ for instructor screens
    • src/screens/shared/ for shared screens
  2. Add the screen to the navigation: 
    // In StudentNavigator.tsx or InstructorNavigator.tsx
<Stack.Screen
  name="MyNewScreen"
  component={MyNewScreen}
  options={{ 
    headerShown: true,
    headerTitle: t('myNewScreen.title'),
  }}
/>
  3. Add translations to src/locales/tr.json and src/locales/en.json

Troubleshooting

Common Issues

Metro Bundler Issues

Problem: Metro bundler not starting or cache issues

Solution:

npm start -- --reset-cache

Or:

npx expo start --clear

iOS Build Issues

Problem: CocoaPods installation fails

Solution:

cd ios
pod deintegrate
pod install
cd ..

Android Build Issues

Problem: Gradle build fails

Solution:

cd android
./gradlew clean
cd ..

TypeScript Errors

Problem: Type errors after updating dependencies

Solution:

npm install
npx tsc --noEmit

Navigation Issues

Problem: Navigation not working or screen not found

Solution:

  • Check that the screen is properly registered in the navigator
  • Ensure screen names match exactly (case-sensitive)
  • Check navigation params are correctly typed

Dark Mode Not Working

Problem: Dark mode not applying correctly

Solution:

  • Check useThemeStore is properly initialized
  • Verify getPalette is called with correct parameters
  • Ensure theme colors are properly defined in theme.ts

Frequently Asked Questions

Q: How do I switch between student and instructor modes?

A: Use the "Switch to Instructor Mode" button in the student profile, or "Switch to Student Mode" in the instructor profile.

Q: Can I use real backend APIs instead of mock data?

A: Yes, replace the MockDataService calls with actual API calls. The service layer is designed to be easily replaceable.

Q: How do I add a new dance category?

A: Add the category to the category filter chips in StudentHomeScreen.tsx and ensure lessons with that category exist in lessons.json.

Q: How do I change the app name and icon?

A: Update app.json with your app name, and replace the icon files in the assets/ folder.

Q: Is Firebase integration included?

A: Firebase integration is planned but not yet implemented. The structure is in place in src/services/firebase/.

Q: How do I enable Stripe payments?

A: The Stripe package is installed but not yet configured. You'll need to add your Stripe keys and implement the payment flow in PaymentScreen.tsx.

Q: Can I customize the color scheme?

A: Yes, edit src/utils/theme.ts to modify colors for both student and instructor themes, including dark mode variants.
